# Calendar-app ChangeLogs

## Initial Build

This update establishes the foundation of the Calendar-app, including core calendar views, local-first data handling, and the first usable event creation flow.

---

## Highlights

* Built the core calendar system (Day, Week, Month, Year views)
* Introduced local-first event storage with change tracking
* Added real event creation UI (no longer placeholder prompts)
* Set up Electron + React + Tailwind development environment
* Established the base component structure for future scaling 

---

## Core Calendar Views

* Added **Month view**

  * Proper calendar grid with spillover days
  * Weekday headers and correct date alignment

* Added **Week view**

  * Timeline layout with hourly structure
  * Events positioned based on time
  * Right-click event creation

* Added **Day view**

  * Vertical timeline with zoom levels
  * Minute-aware event placement
  * Sticky zoom controls and improved scaling behavior

* Added **Year view**

  * 12-month overview grid
  * Each month rendered as a compact mini calendar
  * Event counts per month
  * Click-to-navigate into month view 

---

## Event Creation & Interaction

* Replaced `prompt()`-based event creation with a real UI
* Introduced event creation drawer (slide-out panel)
* Enabled time-based event creation from views
* Connected UI to persistent local event data store 

---

## Local-First Data Foundation

* Implemented local event storage system
* Added append-only change log for future sync support
* Created IPC bridge between renderer and data layer
* Events now persist instead of resetting on reload 

---

## Layout & Navigation

* Built initial header structure with:

  * navigation controls
  * centered title
  * view switcher (Day / Week / Month / Year)
  * Add event action

* Introduced shared calendar header component

* Centralized “Today’s schedule” into reusable control across views

* Moved secondary content (Upcoming, About) into popups/drawers 

---

## UI & Styling

* Added Tailwind CSS support for renderer
* Introduced base styling system:

  * glass-style cards
  * timeline layouts
  * popup/drawer UI
* Added responsive layout behavior
* Improved zoom interaction and control placement in Day view 

---

## App Structure

* Migrated to proper Electron Forge + React setup

* Fixed startup and preload issues

* Organized renderer into structured components:

  * Header
  * Introduction/setup
  * View components (Day, Week, Month, Year)

* Introduced shared utilities:

  * calendar date helpers
  * centralized header
  * shared schedule control 

---

## User Setup

* Added intro/setup screen
* Auto-detects timezone (IANA format)
* Stores user name and timezone locally
* Enables personalized schedule greeting

## Notes

* This is the **first functional foundation** of the app
* Focus is on structure, not final UX polish
* Sets up architecture for future features like:

  * syncing
