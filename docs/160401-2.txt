**Hosted Sync in the Electron App**
After the backend scaffold, I added hosted-mode support in the desktop app so it can actually use that server. The new main-process client is [Application/src/security/hosted-sync-service.js](/C:/Users/REDKING/Projects/Calendar-app/Application/src/security/hosted-sync-service.js). It handles:
- secure backend URL normalization
- browser-based owner sign-in start
- auth-flow polling
- encrypted local storage of backend access/refresh tokens
- access-token refresh
- signed sync request headers
- manual push/pull sync
- local application of pulled event envelopes
- safe disconnect/logout

I integrated that into [Application/src/data/calendar-store.js](/C:/Users/REDKING/Projects/Calendar-app/Application/src/data/calendar-store.js), added IPC/preload methods in [calendar-ipc.js](/C:/Users/REDKING/Projects/Calendar-app/Application/src/ipc/calendar-ipc.js) and [preload.js](/C:/Users/REDKING/Projects/Calendar-app/Application/src/preload.js), and exposed it in the UI through [HostedSyncPanel.jsx](/C:/Users/REDKING/Projects/Calendar-app/Application/src/renderer/components/HostedSyncPanel.jsx), [AboutDrawer.jsx](/C:/Users/REDKING/Projects/Calendar-app/Application/src/renderer/components/AboutDrawer.jsx), [HeroCard.jsx](/C:/Users/REDKING/Projects/Calendar-app/Application/src/renderer/components/HeroCard.jsx), and [renderer.jsx](/C:/Users/REDKING/Projects/Calendar-app/Application/src/renderer/renderer.jsx).
