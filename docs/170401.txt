**Holiday import reliability**
When the holiday API failed, the app was crashing with `Failed to fetch`. I changed the setup/save flow so:
- settings still save
- holiday failure becomes a warning
- the app does not crash
- the setup flow can continue

After that, I broadened holiday coverage by moving holiday resolution into a hybrid provider system:
- primary source: `date-holidays`
- fallback source: Nager.Date

That work moved holiday logic out of the renderer and into the main/store side so the renderer no longer directly fetched holiday data for the main import flow. I added IPC and preload support for that.

I also added a small attribution note for the holiday dataset licensing.
