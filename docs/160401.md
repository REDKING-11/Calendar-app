# Calendar-app ChangeLogs

This update focuses on improving core usability, event handling, and overall structure of the app, while making the UI more customizable and consistent.

---

## Highlights

* Sidebar is now fully functional (filters, search, navigation)
* Full event lifecycle added (create, edit, overview, delete)
* Event system expanded with types, tags, colors, and descriptions
* Major renderer refactor into modular components
* Improved layout with proper sidebar + content shell
* Quick filters now actually affect event visibility

---

## Calendar & Events

* Added event overview popover with edit and delete actions
* Events are now clickable across Day, Week, and Month views
* Added descriptions to events (stored + displayed)
* Introduced event types:

  * Event
  * Task
  * Appointment
* Tasks now support:

  * completion state
  * repeat options
  * deadlines
  * grouping
* Event composer redesigned:

  * scrollable form
  * sticky action footer
  * more compact layout

---

## Sidebar & Filtering

* Sidebar is now functional instead of placeholder UI
* Create button opens event composer
* Mini calendar navigation works (month switching + day select)
* Search filters events by title and tags
* Tag filters now affect visible events globally
* Added Quick Filters:

  * All
  * Today
  * This week
  * This month
* Clear action resets all filters
* Visible events now reflect current view scope (day/week/month/year)

---

## Tags, Colors & Data

* Added persistent event colors (not just visual)
* Added color presets + full color picker
* Introduced reusable shared tag system
* Tags now:

  * can be reused across events
  * have colors
  * are visible in calendar views
* Extended data model to support:

  * tags
  * descriptions
  * event types
  * task-specific fields

---

## Layout & UX

* Rebuilt app shell into proper two-column layout:

  * left = sidebar
  * right = header + calendar
* Sidebar now behaves correctly on desktop (full height)
* Improved mini calendar visuals:

  * stronger highlight for today
  * softer highlight for selected day
* Sidebar scrolling improved:

  * only event list scrolls
* Left-side scrollbar added for event list
* General visual polish for:

  * tag chips
  * color swatches
  * composer layout

---

## Internal Changes

* Split large `renderer.jsx` into smaller components:

  * Sidebar
  * CalendarViewport
  * EventComposerDrawer
  * UpcomingPopover
  * AboutDrawer
* Centralized shared UI:

  * Calendar header
  * Today’s schedule popup
* Moved event draft logic into dedicated helpers
* Improved separation of UI and data logic
* Added compile/syntax verification after major changes
