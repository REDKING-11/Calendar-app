What I changed there over several iterations:

1. Initial year-picker support
- removed the old wheel-based whole-header month switching
- split the old combined month/year label into separate month and year controls
- added `isYearPickerOpen`
- added a generated year list around the current year
- let clicking year select a different year

2. Combined control UI
- merged month and year into one shared visual bar
- added a divider between them
- kept them as separate interactive regions

3. Control sizing / rounded styling fixes
- tightened the control width so month wasn’t leaving excessive blank space
- made the shared wrapper rounded
- removed wrapper styles that were visually creating a square/cube over the rounded shell
- made only the real control visible instead of wrapper divs influencing shape

4. Arrow button styling
- replaced plain text arrows with styled circular arrow buttons

5. Restored separate wheel behavior
- month wheel changes month
- year wheel changes year
- year list itself remains scrollable

6. Dropdown cleanup
- fixed the month dropdown being crushed into a narrow broken column
- fixed the year popup not opening properly
- rebuilt the popup widths and alignment so they don’t inherit tiny trigger sizes

7. CSS recode for the shared control

8. Year button visual alignment
- you pointed out the year segment looked visually stuck “in the wall”
